## Summary

Ran the test-doctor agent workflow on bench-utils. Found and fixed 4 pre-existing bugs, then added comprehensive test coverage to 6 previously-untested modules.

## Bugs Fixed

1. **Test isolation bug** in `bench_cfg::test::test_bench_cfg` - modified the global `BenchCfg` mutex without restoring. Caused cascading failures in `bench_out` and `comp` tests when `test_bench_cfg` ran first.

2. **Fragile relative assertions** in `bench_out::test::test_bench_out_student` - Two `rel_approx_eq!` assertions for near-zero t-statistics failed due to division by tiny denominators. Changed to `approx_eq!` (absolute).

3. **Uncleanable panic tests** in `comp.rs` - `#[should_panic]` tests that used `catch_unwind` but didn't restore the global config after catching. Used `std::panic::catch_unwind` to safely restore config.

4. **Test value truncation** in `latency::test::test_latency_as_f64` - Used a 0.5ms duration which truncated to 0 in Millis. Fixed to use exact unit boundaries.
